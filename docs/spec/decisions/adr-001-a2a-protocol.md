# ADR-001: 에이전트 간 통신 표준으로 A2A 프로토콜 채택

- **상태**: 채택
- **일자**: 2026-04-03

## 맥락

다양한 환경에 배포된 AI Agent들이 서로 통신할 수 있는 표준 프로토콜이 필요했다. Agent마다 다른 프레임워크, 언어, 배포 환경을 사용하므로 벤더 중립적인 프로토콜이 요구되었다.

## 선택지

1. **Google A2A (Agent2Agent)**: Agent 간 통신에 특화된 오픈 프로토콜. HTTP 기반, Agent Card로 서비스 발견, 표준 태스크 관리.
2. **Anthropic MCP (Model Context Protocol)**: AI 앱이 외부 서비스(API, 데이터 소스, 툴)와 소통하는 표준화 레이어. Agent↔툴 연결에 초점.
3. **자체 프로토콜**: 직접 설계. 완전한 제어권.

## 결정

**A2A 프로토콜**을 채택한다.

## 이유

- A2A는 **Agent↔Agent 협업**에 특화되어 있어 이 프로젝트의 목적과 정확히 일치
- MCP는 Agent↔툴 연결에 초점이라 Agent 간 통신에는 부적합
- 자체 프로토콜은 설계/유지보수 비용이 크고, 생태계 호환성이 없음
- A2A는 Agent Card, 태스크 관리, `context_id` 기반 멀티턴 대화를 표준으로 제공
- 150개 이상 조직이 지지하는 오픈 프로토콜로 생태계 확장 가능

## 결과

- 모든 Agent 서버는 A2A 스펙의 엔드포인트를 노출해야 함
- Agent Card (`/.well-known/agent.json`)가 서비스 발견의 기본 단위
- 멀티턴 대화는 A2A 표준 `context_id`로 처리
- MCP는 개별 Agent 내부에서 툴 연결용으로 별도 사용 가능 (A2A와 보완적)
