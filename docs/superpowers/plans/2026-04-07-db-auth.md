# MongoDB/Redis 프로덕션 인증 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 프로덕션 K8s 환경에서 MongoDB root 인증과 Redis requirepass를 활성화한다.

**Architecture:** base K8s manifest는 변경하지 않고, prod overlay에서 JSON patch로 인증 설정을 추가한다. 동일한 `.env.prod` 파일에서 `auth-secrets`(core ns)와 `data-secrets`(data ns)를 각각 생성하여 네임스페이스 분리를 유지한다.

**Tech Stack:** Kustomize JSON patch, K8s Secret, MongoDB 7 `MONGO_INITDB_*`, Redis `--requirepass`, Spring Boot env var binding

---

### Task 2: Prod overlay 인증 패치

**Files:**
- Modify: `infra/k8s/overlays/prod/kustomization.yaml`

이 Task가 핵심이다. 3가지 리소스에 패치를 추가한다:
1. MongoDB StatefulSet — root 계정 환경변수
2. Redis Deployment — `--requirepass` args
3. Auth Deployment — MongoDB 인증 URI + Redis password

- [ ] **Step 1: MongoDB StatefulSet 인증 패치 추가**

`patches` 배열에 MongoDB StatefulSet 대상 패치를 추가한다. `data-secrets` Secret에서 root 계정 정보를 주입.

```yaml
  - target:
      kind: StatefulSet
      name: mongodb
      namespace: data
    patch: |
      - op: add
        path: /spec/template/spec/containers/0/env
        value:
          - name: MONGO_INITDB_ROOT_USERNAME
            valueFrom:
              secretKeyRef:
                name: data-secrets
                key: MONGO_ROOT_USERNAME
          - name: MONGO_INITDB_ROOT_PASSWORD
            valueFrom:
              secretKeyRef:
                name: data-secrets
                key: MONGO_ROOT_PASSWORD
```

- [ ] **Step 2: Redis Deployment 인증 패치 추가**

Redis 컨테이너에 `--requirepass` args를 추가한다. `$(REDIS_PASSWORD)`는 K8s가 같은 컨테이너의 env에서 치환한다.

```yaml
  - target:
      kind: Deployment
      name: redis
      namespace: data
    patch: |
      - op: add
        path: /spec/template/spec/containers/0/env
        value:
          - name: REDIS_PASSWORD
            valueFrom:
              secretKeyRef:
                name: data-secrets
                key: REDIS_PASSWORD
      - op: add
        path: /spec/template/spec/containers/0/args
        value:
          - "--requirepass"
          - "$(REDIS_PASSWORD)"
```

- [ ] **Step 3: Auth Deployment DB credential 패치 추가**

기존 auth 패치 블록(이미 `APP_ENVIRONMENT`, `SPRING_PROFILES_ACTIVE`를 추가하는)에 3개 env var를 추가한다:
- `MONGO_ROOT_USERNAME` — Secret에서 참조
- `MONGO_ROOT_PASSWORD` — Secret에서 참조
- `SPRING_DATA_MONGODB_URI` — 인증 포함 URI (base의 동일 이름 env보다 뒤에 추가되므로 override됨)
- `SPRING_DATA_REDIS_PASSWORD` — Secret에서 참조

기존 auth 패치에 아래 항목을 **추가** (기존 `APP_ENVIRONMENT`, `SPRING_PROFILES_ACTIVE` 뒤에):

```yaml
      - op: add
        path: /spec/template/spec/containers/0/env/-
        value:
          name: MONGO_ROOT_USERNAME
          valueFrom:
            secretKeyRef:
              name: auth-secrets
              key: MONGO_ROOT_USERNAME
      - op: add
        path: /spec/template/spec/containers/0/env/-
        value:
          name: MONGO_ROOT_PASSWORD
          valueFrom:
            secretKeyRef:
              name: auth-secrets
              key: MONGO_ROOT_PASSWORD
      - op: add
        path: /spec/template/spec/containers/0/env/-
        value:
          name: SPRING_DATA_MONGODB_URI
          value: "mongodb://$(MONGO_ROOT_USERNAME):$(MONGO_ROOT_PASSWORD)@mongodb.data.svc.cluster.local:27017/bara-auth?authSource=admin"
      - op: add
        path: /spec/template/spec/containers/0/env/-
        value:
          name: SPRING_DATA_REDIS_PASSWORD
          valueFrom:
            secretKeyRef:
              name: auth-secrets
              key: REDIS_PASSWORD
```

- [ ] **Step 2: MongoDB 기존 데이터 확인**

```bash

```

- 데이터가 없거나 초기화해도 되는 경우: PVC 삭제 후 재생성
  ```bash
  kubectl delete statefulset mongodb -n data
  kubectl delete pvc mongodb-data-mongodb-0 -n data
  kubectl apply -k overlays/prod/
  ```

- 데이터를 보존해야 하는 경우: 수동으로 계정 생성
  ```bash
  kubectl exec -it mongodb-0 -n data -- mongosh --eval "
    db.getSiblingDB('admin').createUser({
      user: '<MONGO_ROOT_USERNAME>',
      pwd: '<MONGO_ROOT_PASSWORD>',
      roles: ['root']
    })
  "
  ```
  이후 MongoDB를 `--auth` 모드로 재시작해야 하므로, StatefulSet에 args를 추가하거나 PVC 삭제 방식이 더 깔끔하다.
